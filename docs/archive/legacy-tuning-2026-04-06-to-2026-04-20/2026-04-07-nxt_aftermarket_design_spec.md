# NXT 애프터마켓 주문 및 파라미터 로직 설계서

## 1. 문서 개요

### 1.1 목적
본 문서는 NXT 애프터마켓 환경에 최적화된 자동매매 봇의 주문 로직, 전략 프로필, 파라미터 정책, 리스크 통제 장치를 정의한다.

정규장 로직을 단순 재사용하지 않고, 애프터마켓의 낮은 유동성, 넓은 스프레드, 이벤트 집중형 체결 구조를 반영한 별도 운영 모드를 설계하는 것을 목표로 한다.

### 1.2 범위
본 설계는 아래 범위를 포함한다.

- NXT 애프터마켓 전용 전략 모드
- 주문 생성 및 체결 정책
- 전략별 파라미터 구조
- 리스크 가드 및 강제 청산 규칙
- AI 판단 프롬프트 분리
- 운영/테스트 기준

### 1.3 비범위
본 문서는 아래 항목은 상세 구현 범위에서 제외한다.

- 증권사 API별 상세 인증/연결 방식
- 실시간 뉴스 공급자 계약 및 인프라 구성
- 백테스트 엔진 전체 구현
- GUI/모니터링 대시보드 상세 화면 설계

## 2. 시장 특성 및 설계 전제

### 2.1 애프터마켓 특성
NXT 애프터마켓은 정규장 대비 유동성이 현저히 낮고, 종목별 호가 공백과 체결 단절이 빈번하다. 따라서 정규장에서 유효한 거래량 돌파 추종 전략은 신뢰도가 낮다.

애프터마켓에서는 아래 특성이 특히 중요하다.

- 거래량이 적고 호가창이 얇다
- 매수/매도 1호가 간 스프레드가 넓게 벌어진다
- 실적 발표, 수주 공시, 가이던스 변경 등 장 마감 후 이벤트에 체결이 집중된다
- 뉴스가 없는 시간에는 방향성보다 스프레드 구조가 더 중요하다
- 해외 지수/미국 프리마켓 움직임이 일부 종목군에 선반영될 수 있다

### 2.2 제도 및 운영 전제
현재 NXT 공식 기준 애프터마켓 체결 구간은 **15:40~20:00**이며, 프리·애프터마켓에서는 **공매도가 금지**되고, 가격제한폭은 **전일 KRX 종가 대비 ±30%**가 적용된다.

또한 시장을 지정하지 않으면 증권사 SOR/최선집행 로직에 따라 KRX 또는 NXT로 자동 배분될 수 있으므로, 본 설계에서는 애프터마켓 전략 실행 시 **거래소를 NXT로 강제 지정**하는 것을 기본 전제로 한다.

### 2.3 핵심 설계 원칙
애프터마켓 모드는 다음 원칙을 따른다.

1. **시장가 주문 금지**
2. **전략 미충족 시 거래하지 않음**
3. **작은 주문, 짧은 보유, 빠른 취소**
4. **이벤트/스프레드/매크로 전략 분리**
5. **정규장과 완전히 다른 파라미터 프로필 사용**
6. **데이터 지연 또는 호가 공백 감지 시 즉시 주문 취소**

## 3. 시스템 목표

### 3.1 목표
- 이벤트 발생 직후 짧은 찰나의 유동성 집중 구간을 포착
- 넓은 스프레드 환경에서 제한적 passive 수익 기회 확보
- 해외 상관자산 급변 시 국내 관련주 단기 반응 선반영 활용

### 3.2 성공 기준

- 체결당 예상 슬리피지 통제 가능

- 비정상 호가/체결 공백 구간에서 자동 거래 중지

## 4. 상위 아키텍처

### 4.1 구성 요소
전체 시스템은 아래 모듈로 구성한다.

- `session_router`
  - 현재 시간이 정규장/프리마켓/애프터마켓 중 어디인지 판별
- `market_profile_manager`
  - 세션별 주문/리스크/전략 파라미터 로드
- `signal_engine`
  - 뉴스, DART 공시, 호가, 체결, 상관자산 신호 통합
- `strategy_selector`
  - 이벤트/스프레드/매크로/노트레이드 중 하나 선택
- `risk_guard`
  - 주문 전 사전 리스크 검증
- `execution_engine`
  - 지정가/공격적 지정가/IOC/FOK 기반 주문 실행
- `position_manager`
  - 보유 포지션 추적, 부분 청산, 강제 손절, 시간 기반 청산
- `ai_engine`
  - 시장 컨텍스트 기반 전략 타입 및 주문 방식 보조 판단
- `monitoring_logger`
  - 신호, 주문, 체결, 취소, 리스크 이벤트 로그 기록

### 4.2 처리 흐름
1. 세션 판별
2. 애프터마켓 프로필 로드
3. 실시간 뉴스/공시/호가/체결 수집
4. 전략 조건 점수화
5. 전략 선택
6. 리스크 사전 검증
7. 주문 생성
8. 짧은 TTL 기반 체결 대기
9. 미체결 시 취소/재호가
10. 청산 또는 강제 종료

## 5. 세션 모드 설계

### 5.1 세션 구분
- 정규장
- NXT 프리마켓
- NXT 애프터마켓
- 비거래 시간

### 5.2 애프터마켓 기본 정책
애프터마켓에서는 기본 상태를 `NO_TRADE`로 두고, 아래 조건 중 하나를 만족할 때만 거래를 허용한다.

- 이벤트 드리븐 신호 활성
- 스프레드 캡처링 신호 활성
- 매크로/페어 신호 활성

### 5.3 의사코드
```python
def select_profile(now, news_signal, spread_signal, macro_signal):
    if not is_nxt_aftermarket(now):
        return "REGULAR_OR_OFF"

    if news_signal.active and now <= "17:20":
        return "AFTER_EVENT_DRIVEN"

    if spread_signal.active and not news_signal.recent_10m:
        return "AFTER_SPREAD_CAPTURE"

    if macro_signal.active and now >= "17:00":
        return "AFTER_MACRO_PAIR"

    return "AFTER_NO_TRADE"
```

## 6. 주문 실행 정책

### 6.1 주문 원칙
애프터마켓에서는 다음 규칙을 강제한다.

- 시장가 주문 사용 금지
- 기본 주문은 지정가
- 급한 진입이 필요할 경우에만 공격적 지정가 사용
- 가능하면 IOC/FOK 사용
- 미체결 주문은 매우 짧은 TTL 이후 취소
- stale 데이터 감지 시 모든 주문 취소

### 6.2 공통 실행 정책
```python
AFTER_MARKET_EXEC_POLICY = {
    "force_venue": "NXT",
    "allow_market_order": False,
    "allow_short_sell": False,
    "default_order_type": "limit",
    "aggressive_limit_offset_ticks": 1,
    "max_order_age_ms": 800,
    "max_requote_count": 3,
    "cancel_if_data_stale_ms": 300,
    "cancel_if_news_feed_stale_ms": 1000,
}
```

### 6.3 주문 타입 정의
- `limit`
  - 일반 지정가
- `aggressive_limit`
  - 반대 최우선 호가 기준 1틱 유리하게 배치하는 지정가
- `ioc_limit`
  - 즉시 체결 가능한 수량만 체결 후 잔량 취소
- `fok_limit`
  - 전량 즉시 체결되지 않으면 전량 취소

### 6.4 재호가 정책
- 최초 주문 미체결 시 취소
- 최대 `max_requote_count` 회까지만 재호가
- 재호가 중 스프레드가 악화되면 즉시 중단
- 이벤트 신호가 약화되면 후속 주문 중단

## 7. 전략 설계

### 7.1 전략 A: 이벤트 드리븐 찌르기

#### 7.1.1 개요
장 마감 후 집중되는 실적 발표, 수주 공시, 가이던스 변경, 대규모 계약, 자사주/소각, 주주환원 관련 이벤트를 실시간 탐지하여 짧게 진입하는 전략이다.

#### 7.1.2 진입 조건
- 뉴스 또는 공시 점수가 기준 이상
- 이벤트 발생 직후 실제 체결대금 증가
- 스프레드가 허용 범위 이내
- 반대편 잔량이 내 주문 대비 충분히 존재
- 이벤트 발생 이후 최초 반응 구간 내

#### 7.1.3 파라미터
```python
AFTER_EVENT_DRIVEN = {
    "window": ["15:40", "17:20"],
    "min_news_score": 0.82,
    "min_10s_trade_value_krw": 50_000_000,
    "max_spread_ticks": 8,
    "max_spread_bps": 40,
    "min_opposite_depth_multiple": 3.0,
    "entry_offset_ticks": 1,
    "max_entry_retries": 2,
    "hard_stop_pct": 0.8,
    "take_profit_pct": 1.2,
    "max_holding_sec": 120,
    "cooldown_sec_after_exit": 180,
}
```

#### 7.1.4 진입 의사코드
```python
if news_score >= 0.82 \
   and trade_value_10s >= 50_000_000 \
   and spread_ticks <= 8 \
   and spread_bps <= 40 \
   and visible_ask_depth_krw >= 3 * my_order_notional:
       place_aggressive_limit_buy(best_ask + 1_tick)
```

#### 7.1.5 청산 규칙
- 목표 수익 도달 시 청산
- 최대 보유 시간 초과 시 청산
- 신호 반전 또는 체결 둔화 시 청산
- 손절 기준 도달 시 즉시 청산

#### 7.1.6 주의 사항
이 전략은 “좋은 뉴스” 자체보다 “뉴스 이후 체결이 따라붙는가”가 더 중요하다. 뉴스만 있고 거래가 붙지 않는 경우는 유동성 함정일 가능성이 높다.

### 7.2 전략 B: 스프레드 캡처링

#### 7.2.1 개요
뉴스가 없는 구간에서 넓게 벌어진 스프레드를 활용하여 양방향 지정가를 소규모로 제시하고, 체결 후 반대편으로 청산하는 전략이다.

#### 7.2.2 적용 조건
- 최근 10분 내 중요 뉴스 없음
- 스프레드가 충분히 큼
- 직전 60초 실현 변동성이 낮음
- 체결 공백이 너무 길지 않음

#### 7.2.3 파라미터
```python
AFTER_SPREAD_CAPTURE = {
    "window": ["16:00", "19:20"],
    "recent_news_block_min": 10,
    "min_spread_ticks": 3,
    "min_spread_bps": 18,
    "max_realized_vol_60s_pct": 0.35,
    "quote_size_ratio_to_visible_depth": 0.10,
    "max_child_order_krw": 3_000_000,
    "inventory_cap_units": 1,
    "adverse_move_stop_ticks": 2,
    "max_quote_age_ms": 1500,
    "max_unfilled_requotes": 5,
}
```

#### 7.2.4 운용 규칙
- 양쪽 호가에 동시에 작은 수량 제시
- 한쪽 체결 시 즉시 반대편 청산 호가로 전환
- 체결 후 불리하게 2틱 이상 이동 시 손절
- 잔량이 줄거나 스프레드가 축소되면 신규 호가 중단

#### 7.2.5 주의 사항
이 전략은 사실상 초소형 인벤토리 기반 passive market making이다. 방향성 예측보다 재고 리스크 통제가 중요하다.

### 7.3 전략 C: 매크로 헷징 및 페어 트레이딩

#### 7.3.1 개요
미국 프리마켓, 나스닥 선물, 섹터 ETF, 대형 리더 종목 움직임을 참고하여 국내 상관 종목군의 후행 반응을 짧게 공략하는 전략이다.

#### 7.3.2 적용 예시
- 나스닥 100 선물 급등
- 엔비디아 프리마켓 급등
- 반도체 섹터 ETF 급등
- 국내 반도체 관련주의 상대적 지연 반응 확인

#### 7.3.3 파라미터
```python
AFTER_MACRO_PAIR = {
    "window": ["17:00", "19:30"],
    "leaders": ["NQ_FUT", "NVDA_US_PM", "SOXX_PM"],
    "min_leader_move_pct": 1.0,
    "min_zscore": 2.0,
    "min_corr_20d": 0.60,
    "target_universe": ["SK하이닉스", "한미반도체", "서울반도체", "ISC"],
    "max_spread_ticks": 6,
    "entry_pullback_atr_ratio": 0.3,
    "take_profit_pct": 0.7,
    "hard_stop_pct": 0.6,
    "overnight_allowed": False,
}
```

#### 7.3.4 진입 원칙
- 리더 자산 급변 확인
- 상관 종목군 필터 통과
- 국내 종목이 이미 과도하게 급등한 경우 추격 금지
- 짧은 눌림 또는 재진입 가능 구간에서만 진입

#### 7.3.5 주의 사항
이 전략은 추격매수보다 “상관자산 반응 지연”을 이용하는 구조여야 한다. 이미 많이 오른 종목을 뒤늦게 추격하면 체결 리스크가 급증한다.

## 8. 공통 리스크 가드

### 8.1 목적
애프터마켓은 체결이 불연속적이므로, 전략 신호보다 리스크 가드가 우선해야 한다.

### 8.2 공통 리스크 파라미터
```python
AFTER_RISK_GUARD = {
    "max_gross_exposure_ratio_vs_regular": 0.35,
    "max_single_name_exposure_krw": 15_000_000,
    "max_child_order_krw": 5_000_000,
    "max_child_vs_opposite_visible_depth": 0.15,
    "daily_loss_limit_krw": 1_500_000,
    "strategy_loss_limit_krw": 500_000,
    "consecutive_loss_limit": 3,
    "kill_if_spread_ticks_gt": 12,
    "kill_if_no_trade_print_sec": 60,
    "kill_if_quote_gap_pct_gt": 1.5,
    "flatten_all_time": "19:50",
}
```

### 8.3 필수 차단 조건
아래 중 하나라도 만족하면 신규 주문을 금지한다.

- 반대편 visible depth가 내 주문의 3배 미만
- 스프레드가 12틱 초과
- 마지막 체결 이후 60초 이상 경과
- 호가 데이터 지연
- 뉴스 피드 지연
- 전략 누적 손실 한도 초과
- 일일 손실 한도 초과
- 동일 전략 연속 손실 횟수 초과

### 8.4 포지션 종료 규칙
- 19:50 이후 전 포지션 강제 청산
- 일일 손실 한도 도달 시 즉시 거래 종료
- 데이터 이상 발생 시 즉시 주문 취소 및 포지션 축소
- 시스템 장애 감지 시 모든 신규 주문 차단

## 9. 파라미터 정책

### 9.1 정규장 파라미터 재사용 금지
정규장 기준 절대값 필터는 애프터마켓에서 그대로 사용할 수 없다. 예를 들어 정규장 기준 `MIN_WINDOW_BUY_VALUE = 20,000,000`은 애프터마켓에서 지나치게 높거나 종목별 왜곡을 일으킬 수 있다.

### 9.2 상대값 기반 설계
애프터마켓 파라미터는 가능하면 종목별 정규장 기준치 대비 상대값으로 정의한다.

```python
min_trade_value = max(
    symbol_regular_1m_median_trade_value * 0.08,
    10_000_000,
)
```

### 9.3 권장 설계 방향
- 절대값 + 상대값 혼합 사용
- 종목군별 별도 프로필 지원
- 이벤트 전략은 체결대금/뉴스 점수 중심
- 스프레드 전략은 스프레드/변동성 중심
- 매크로 전략은 상관성/z-score 중심

### 9.4 파라미터 관리 방식
- `yaml` 또는 `json` 기반 프로필 분리
- 전략별/세션별/종목군별 계층 구조
- 운영 중 hot reload 가능하면 이상적
- 모든 변경 내역은 로그로 기록

## 10. AI 판단 프롬프트 분리

### 10.1 목적
정규장용 AI 판단 규칙은 애프터마켓에 부적합하므로, 별도 프롬프트를 둔다.

### 10.2 프롬프트 예시
```python
NXT_AFTERMARKET_PROMPT = """
현재 시장은 NXT 애프터마켓이다.
정규장과 달리 거래량 돌파 추종의 신뢰도가 낮다.
다음 우선순위로 판단하라:

1) 장 마감 후 공시/실적/수주/가이던스 변화 등 이벤트 존재 여부
2) 이벤트 직후 실제 체결대금 증가와 호가 스프레드 축소 여부
3) 뉴스가 없으면 방향성 추종보다 스프레드 캡처 가능성
4) 미국 프리마켓/나스닥 선물/섹터 ETF 움직임이 국내 관련주에 전이되는지
5) 호가가 비거나 체결이 끊기면 무조건 보수적으로 판단

금지:
- 시장가 주문 제안
- 얇은 호가를 무시한 추격 진입
- 뉴스 없는 돌파 추종
- 손절 없는 양방향 호가 제안

출력:
- 전략 타입(event / spread / macro / no-trade)
- 진입 가격 방식(limit / aggressive-limit)
- 허용 최대 슬리피지
- 예상 보유 시간
- 즉시 취소 조건
"""
```

### 10.3 AI 출력 사용 원칙
AI 출력은 단독 실행 조건이 아니라 보조 판단값으로 사용한다.

- AI 출력만으로 주문 금지
- rule-based 필터 통과 후에만 참고
- 리스크 가드보다 우선할 수 없음

## 11. 예외 처리 및 장애 대응

### 11.3 시스템 이상
- 프로세스 재시작 후 미확인 주문 상태 복구
- 브로커 잔고/체결 대사 수행
- 장애 복구 전 신규 진입 차단

## 13. 테스트 계획

### 13.1 단위 테스트
- 세션 분기 테스트
- 전략 선택 조건 테스트
- 주문 가격 계산 테스트
- 손절/익절 계산 테스트
- stale 데이터 차단 테스트

### 13.2 시뮬레이션 테스트
- 이벤트 직후 체결 증가 시나리오
- 뉴스 없이 스프레드만 벌어진 시나리오
- 체결 단절 시나리오
- 급격한 반대 방향 움직임 시나리오
- 데이터 지연/누락 시나리오

### 13.3 리플레이 테스트
과거 애프터마켓 호가/체결/공시 데이터를 재생하여 아래를 검증한다.

- 전략별 진입 빈도
- 체결 가능성
- 슬리피지 분포
- 강제 손절 빈도
- 일일 손실 한도 발동 빈도

### 13.4 모의운영
실거래 전 최소 2단계 모의운영을 수행한다.

1. 신호만 생성하고 주문 미발송
2. 소액 실거래 또는 테스트 계정 기반 제한 운영

## 15. 결론

NXT 애프터마켓 자동매매는 정규장처럼 거래량 돌파를 추격하는 구조보다, 아래 3가지 상황에만 선택적으로 반응하는 구조가 적합하다.

- 이벤트 직후의 체결 집중
- 뉴스 부재 구간의 넓은 스프레드
- 해외 상관자산 급변에 대한 국내 종목군의 지연 반응

따라서 시스템의 핵심은 “항상 거래하는 봇”이 아니라, **거래하지 않는 시간을 기본값으로 두고 제한된 유동성 구간에서만 작고 짧게 진입하는 봇**이어야 한다.

또한 애프터마켓에서는 전략의 정교함보다도 아래 4가지가 더 중요하다.

- 시장가 금지
- 짧은 주문 수명
- 보수적 포지션 크기
- 강한 리스크 차단 규칙

이 문서의 설계를 기준으로 구현할 경우, 정규장 엔진과 분리된 애프터마켓 전용 프로필을 안전하게 운영할 수 있다.
